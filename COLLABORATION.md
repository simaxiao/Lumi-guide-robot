# COLLABORATION｜Lumi 路明协作机制说明

Lumi 路明是一个面向**视障出行场景**的开放共创导盲机器人 Demo 项目。

为了让项目能够长期、稳定、健康地推进，  
当前阶段我们采用：

**开放共创、角色分层、边界清晰、逐步推进** 的协作方式。

---

## Why This File Exists

对于一个仍处于 Demo 验证阶段的开放共创项目来说，  
不同参与者的投入方式、责任范围、参与周期和预期回报并不相同。

如果一开始把所有人都统一叫做“合伙人”或“联合创始人”，很容易出现以下问题：

- 贡献大小与身份定义不匹配
- 参与深度不同，但预期相同
- 项目推进过程中职责模糊
- 后续权益、署名与决策边界不清晰

因此，Lumi 当前不会默认把所有参与者都定义为“合伙人”。  
我们更重视的是：

- 是否真正参与推进
- 是否对具体模块负责
- 是否理解项目当前阶段
- 是否愿意在清晰边界内长期共创

---

## Current Stage

Lumi 当前仍处于 **Demo 验证阶段**。

现阶段目标包括：

- 完成硬件原型
- 完成基础运动控制
- 完成手机通信
- 完成最低限度安全制动
- 完成可演示场景验证

因此，我们当前最重要的不是做“大团队”，  
而是搭建一个足够轻、足够清晰、足够可信的共创机制，  
先把第一台样机做出来。

---

## Collaboration Roles

当前合作关系主要分为 4 类：

### 1. 核心共创成员

**定义：**  
愿意长期参与项目推进，并对具体模块结果负责的核心参与者。

**典型特点：**
- 长周期参与
- 每周有相对稳定的投入时间
- 负责某一模块的具体推进与交付
- 会进入核心项目协作节奏

**适合的方向包括：**
- 电子 / 嵌入式
- 结构 / 机械
- 控制 / 机器人运动
- 样机整合
- 内容记录与传播运营等

**特别说明：**  
核心共创成员不等于自动成为“联合创始人”。  
如果未来项目进入正式立项、注册主体、融资或商业化推进阶段，再根据实际贡献、持续投入和责任范围讨论更长期合作机制。

---

### 2. 顾问 / 导师

**定义：**  
愿意从专业视角给予建议、指导和经验支持的外部帮助者。

**典型特点：**
- 提供技术建议、方向判断或资源建议
- 不参与具体模块交付
- 不进入高频执行节奏
- 以阶段性交流或关键节点支持为主

**可能包括：**
- 硬件专家
- 机器人方向从业者
- 供应链老兵
- 医疗辅助器械相关人士
- 公益助残领域经验人士
- 产品、设计、品牌传播方向导师

**特别说明：**  
顾问 / 导师以“建议支持”为主，不默认绑定项目日常管理、交付和决策义务。

---

### 3. 供应链 / 器件支持伙伴

**定义：**  
为项目提供器件、样机、打样、加工、生产相关支持的合作方或资源伙伴。

**典型特点：**
- 提供器件、模块、加工或样机支持
- 帮助提升打样效率与工程可落地性
- 不深度参与研发日常节奏
- 更偏资源协作与工程支持

**支持方向可能包括：**
- BLDC 电机 / 减速器
- 电机驱动板
- MCU / 开发板
- IMU / ToF / 超声波模块
- 电池与充电模组
- 电磁制动器
- CNC / 3D 打印 / 结构件加工
- 小批量 PCBA 打样
- 地图 API / 云服务 credits

**特别说明：**  
供应链 / 器件支持伙伴以“资源支持 / 工程支持 / 供应支持”为主，  
不默认等同于核心研发成员，也不自动进入项目核心决策层。

---

### 4. 公益 / 场景验证伙伴

**定义：**  
帮助项目连接真实使用场景、用户反馈和社会价值验证的合作伙伴。

**典型特点：**
- 提供真实场景反馈
- 协助测试体验与安全边界
- 参与需求理解、场景观察与验证
- 不负责技术研发交付

**可能包括：**
- 盲协
- 助残公益组织
- 志愿者团队
- 测试支持方
- 无障碍研究或服务机构
- 相关医院 / 康复机构 / 社工组织等

**特别说明：**  
当前阶段涉及的所有测试与体验验证，都应以安全、谨慎、边界清晰为前提。  
Demo 阶段的验证重点是场景理解、流程观察和体验反馈，不等于正式产品交付。

## How We Work

Lumi 当前阶段的协作原则如下：

### 1. 先共识，再开工
在正式开始协作前，我们希望每一位参与者都能理解：
- 项目当前还处于 Demo 阶段
- 项目不是短期商业化招聘
- 当前以开放共创、公益探索和样机验证为主
- 项目节奏需要现实、克制、可持续

### 2. 先小范围，再扩大
我们不会一开始就组建很大的团队。  
会先围绕 Demo 所需的核心模块，逐步确认首批真正能推进事情的人。

### 3. 先明确责任，再承诺协作
每一位核心共创成员，都应尽量有清晰的模块边界与责任范围。  
不鼓励“泛参与但无人负责”的协作方式。

### 4. 沟通比头衔更重要
当前阶段最重要的不是头衔，而是：
- 是否能认真沟通
- 是否理解项目边界
- 是否愿意长期推动
- 是否愿意对结果负责

## What We Value

我们更看重以下特质：

- 真诚
- 责任感
- 长期主义
- 沟通清晰
- 尊重边界
- 愿意一起从 0 到 1 做事

比起“先拿身份”，  
我们更认同“先一起把事情做出来”。

---

## What Participants Can Expect

参与 Lumi 当前阶段的开放共创，你可以期待：

### 1. 真实的 0-1 项目经验
从产品定义、硬件打样、控制验证、结构优化，到协作推进和公开展示，这会是一次真正从想法走向 Demo 的实践过程。

### 2. 一件有公益价值、也有趣的事
如果能用机器人和 AI，为视障出行这个问题做出一点点真正有意义的尝试，这本身就是一件很酷的事。

### 3. 未来继续参与的可能性
如果未来项目逐步成熟，进入更正式的组织化、公司化或商业化阶段，早期深度参与者将拥有优先继续参与的机会。

---

## Final Note

Lumi 当前是一个以 **共创、公益、验证** 为核心的开放项目。

我们欢迎不同背景的人加入，  
但也希望大家在一开始就对合作关系保持清晰理解：

**不是所有参与者都需要叫“合伙人”，  
真正重要的是，大家是否在各自的位置上，认真推动这件事向前。**

如果你认同这样的协作方式，欢迎加入 Lumi 路明。
